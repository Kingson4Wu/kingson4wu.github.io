# Rewrite Rules

Use this for old-post modernization and AI-tone cleanup.

## Rewrite Priorities

Apply in this order:

1. Preserve factual meaning, personal viewpoint, and technical conclusion.
2. Identify the article's central claim.
3. Rebuild structure around that claim.
4. Replace AI-style or note-like text with concrete reasoning.
5. Polish sentences last.

## What to Preserve

- Frontmatter.
- Title unless the user requests title work.
- Dates, tags, slugs, images, links, code blocks, commands, and references.
- Technical terms, constraints, and conclusions.
- The author's direct judgment, even if it is mildly sharp.
- Personal stories and real work context.

## What to Change

- Convert bullet piles into paragraphs when the bullets are not true steps or categories.
- Add section headings that reveal the argument.
- Merge duplicate points.
- Replace generic statements with scenario-specific reasoning.
- Change "AI report style" into first-person technical judgment.
- Rework endings that merely summarize the topic into endings that state the author's takeaway.

## AI-Tone Cleanup

Watch for these patterns:

- "随着 X 的快速发展..."
- "这是一把双刃剑..."
- "既带来了机遇，也带来了挑战..."
- "我们应该积极拥抱变化..."
- "未来依然充满希望..."
- "总的来说，这说明..."
- "在当今快速变化的时代..."
- balanced paragraphs that never state the author's actual view.

Replace them with:

- a concrete trigger,
- the author's direct conclusion,
- a real cost or benefit,
- a practical next step,
- a boundary condition.

## Old-Post Modernization

Older posts often read like personal knowledge-base notes. Keep the useful density, but make them publishable:

- Add a short lead explaining why the note matters.
- Promote only true major ideas to headings.
- Keep checklists when the article is intended as a checklist.
- For reading notes, reduce quote-heavy sections and add the author's interpretation.
- For technical summaries, distinguish copied/reference material from personal understanding.

## Ending Rules

Good endings:

- name the author's current judgment,
- say where the idea applies,
- warn about a practical pitfall,
- state what the author would do next time.

Bad endings:

- motivational slogan,
- generic AI summary,
- "希望对大家有帮助",
- a Reference section as the only ending when the article has its own conclusion.
